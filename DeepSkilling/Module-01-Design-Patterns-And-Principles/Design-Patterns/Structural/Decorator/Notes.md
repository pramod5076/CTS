# Decorator Pattern

## Definition

The Decorator Pattern adds new functionality to an object dynamically without modifying its structure.

It wraps an existing object and extends its behavior.

## Why Use Decorator?

Inheritance can create many subclasses.

Decorator provides a flexible alternative by adding features at runtime.

## Advantages

* Flexible functionality extension
* Follows Open Closed Principle
* Reduces subclass explosion

## Real World Example

Ordering coffee:

Base Coffee

* Milk

* Sugar

* Chocolate

Each add-on decorates the original coffee.
