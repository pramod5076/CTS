# Adapter Pattern

## Definition

The Adapter Pattern allows two incompatible interfaces to work together.

It acts as a bridge between existing code and new code without modifying either side.

## Why Use Adapter?

Sometimes we need to integrate third-party libraries or legacy systems whose interfaces do not match our application's requirements.

Instead of changing existing code, we create an adapter.

## Advantages

* Improves compatibility
* Encourages code reuse
* Reduces modifications to existing code

## Real World Example

A laptop charger adapter allows a laptop plug to fit different power sockets.
