# Observer Pattern

## Definition

The Observer Pattern establishes a one-to-many relationship between objects.

When one object changes its state, all dependent objects are automatically notified.

## Why Use Observer?

It allows objects to communicate without being tightly coupled.

Commonly used in:

* Event systems
* Notification systems
* Stock market applications
* Chat applications

## Advantages

* Loose coupling
* Automatic updates
* Easy scalability

## Real World Example

You subscribe to a YouTube channel.

Whenever a new video is uploaded, all subscribers receive notifications.
